\section{Indicaciones generales}
En cada ejercicio separar orden físico, orden lógico y causalidad. Cuando uses relojes de Lamport o vector clocks, indicar qué garantía dan y qué no pueden concluir.

\section{Ejercicios}
\begin{uaexercise}\textbf{1.} Explicar por qué dos timestamps físicos de nodos distintos no alcanzan para decidir causalidad entre eventos.\end{uaexercise}
\begin{uaexercise}\textbf{2.} Dado dos procesos con eventos locales y dos mensajes, marcar las relaciones happened-before y señalar qué eventos son concurrentes.\end{uaexercise}
\begin{uaexercise}\textbf{3.} Aplicar las reglas de relojes de Lamport a una secuencia de eventos con envío y recepción de mensajes. Mostrar los contadores resultantes.\end{uaexercise}
\begin{uaexercise}\textbf{4.} Dar un ejemplo donde $L(a) < L(b)$ pero no pueda concluirse que $a \rightarrow b$.\end{uaexercise}
\begin{uaexercise}\textbf{5.} Comparar dos vector clocks e indicar si representan causalidad en algún sentido o concurrencia: $(2,1,0)$ y $(1,2,0)$; $(1,2,0)$ y $(2,3,0)$.\end{uaexercise}
\begin{uaexercise}\textbf{6.} Explicar por qué vector clocks detectan concurrencia y Lamport no.\end{uaexercise}
\begin{uaexercise}\textbf{7.} En un store eventualmente consistente, dos réplicas aceptan escrituras concurrentes sobre el mismo objeto. Proponer cómo usar vector clocks para decidir si se sobrescribe o se reconcilia.\end{uaexercise}
\begin{uaexercise}\textbf{8.} Identificar el trade-off de usar vector clocks en un sistema con muchos nodos.\end{uaexercise}
