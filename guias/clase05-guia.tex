\section{Nivel 1: fundamentos}

1. Definir formalmente ACID y explicar el rol de cada propiedad.  
2. Explicar qué significa aislamiento en términos de observabilidad.  
3. Diferenciar atomicidad de aislamiento con ejemplos concretos.  
4. Dar un ejemplo donde atomicidad se cumple pero aislamiento falla.  

\section{Nivel 2: serializabilidad}

5. Definir serializabilidad formalmente.  
6. Construir un grafo de precedencia para dos transacciones con conflictos RW.  
7. Dar un ejemplo de ejecución no serializable y demostrarlo con el grafo.  
8. Explicar por qué un ciclo implica imposibilidad de orden secuencial.  

\section{Nivel 3: anomalías}

9. Construir un ejemplo de dirty read.  
10. Construir un ejemplo de non-repeatable read.  
11. Construir un ejemplo de phantom read.  
12. Construir un ejemplo completo de write skew.  
13. Explicar por qué write skew no genera conflicto directo.  

\section{Nivel 4: aislamiento}

14. Comparar Read Committed vs Repeatable Read.  
15. Explicar Snapshot Isolation con un ejemplo.  
16. Mostrar una ejecución válida en SI pero no serializable.  

\section{Nivel 5: commit distribuido}

17. Describir paso a paso 2PC con 3 nodos.  
18. Construir escenario donde coordinador falla en 2PC.  
19. Explicar por qué los participantes quedan bloqueados.  
20. Describir cómo 3PC intenta resolver esto.  

\section{Nivel 6: integración}

21. Relacionar transacciones con consenso.  
22. Explicar por qué 2PC no resuelve consenso.  
23. Diseñar sistema que combine consenso + transacciones.  
24. Aplicar modelo D al problema.
