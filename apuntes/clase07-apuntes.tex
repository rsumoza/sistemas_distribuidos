\section{Introducción}

El paradigma moderno de sistemas distribuidos se basa en \textbf{event sourcing} y \textbf{logs inmutables}. En lugar de mutar estado directamente, los sistemas registran eventos que luego pueden ser reproducidos.

\begin{quote}
El estado es una función del log: $State = f(Log)$
\end{quote}

\section{Commit Log}

Un commit log es una secuencia ordenada e inmutable:

\[
L = (e_0, e_1, e_2, \dots)
\]

Cada evento $e_i$ tiene un offset $i$.

\subsection{Propiedades}
\begin{itemize}
\item \textbf{Inmutabilidad}: no se modifica, solo append
\item \textbf{Orden}: total dentro de la partición
\item \textbf{Reproducibilidad}: permite reconstruir estado
\end{itemize}

\section{Kafka}

\subsection{Modelo}

Un topic se divide en particiones:

\[
Topic = \{P_0, P_1, ..., P_k\}
\]

Cada partición es un log independiente.

\subsection{Orden}

El orden está garantizado solo dentro de cada partición.

\subsection{Replicación}

Cada partición tiene:
\begin{itemize}
\item líder (acepta writes)
\item seguidores (replican)
\end{itemize}

\section{Offsets}

Los offsets permiten:
\begin{itemize}
\item control de progreso
\item relectura (replay)
\item tolerancia a fallas
\end{itemize}

\section{Semánticas de entrega}

\subsection{At-most-once}

Commit antes de procesar:

\begin{itemize}
\item pérdida posible
\item sin duplicación
\end{itemize}

\subsection{At-least-once}

Commit después de procesar:

\begin{itemize}
\item duplicación posible
\item sin pérdida
\end{itemize}

\subsection{Exactly-once}

Requiere:

\begin{itemize}
\item idempotencia (procesamiento repetible)
\item control transaccional de offsets
\item coordinación productor-consumidor
\end{itemize}

\section{Streaming}

\subsection{Modelo}

Procesamiento continuo:

\begin{itemize}
\item eventos entrantes
\item transformaciones
\item salidas
\end{itemize}

\subsection{Stateful streaming}

Ejemplo:

\begin{itemize}
\item conteo de eventos por ventana
\item joins entre streams
\end{itemize}

\section{Fallas y recuperación}

\begin{itemize}
\item duplicación: retry
\item pérdida: commit temprano
\item reorder: múltiples particiones
\end{itemize}

\section{Conclusión}

\begin{quote}
El log es la base de sistemas modernos; los servicios son transformaciones del log.
\end{quote}
