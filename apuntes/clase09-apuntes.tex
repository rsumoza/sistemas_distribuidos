\section{Introducción}
La clase 09 desarrolla el tema ``Logs y procesamiento distribuido'' como una extensión directa del framework del curso. El foco no está puesto solamente en describir una técnica, sino en entender qué problema distribuye, qué hipótesis de falla la vuelven necesaria, qué garantía permite ofrecer y qué costo operativo introduce.

\section{Desarrollo conceptual}
Modelar sistemas como flujos de eventos sobre logs particionados y consumidores coordinados. Este tema debe pensarse siempre sobre un caso concreto: una operación, una lectura, una réplica, un log o un servicio remoto que necesita coordinarse con otros elementos del sistema. La clase discute cómo cambia la decisión de diseño cuando cambian las hipótesis de falla o las garantías exigidas por el dominio.

\subsection{Preguntas guía}
\begin{itemize}
\item ¿Cómo aparece topics en un sistema real? \item ¿Cómo aparece partitions en un sistema real? \item ¿Cómo aparece offsets en un sistema real? 
\item ¿qué síntoma observaría un usuario o un operador?
\item ¿qué concesión de rendimiento, complejidad o disponibilidad introduce la solución?
\end{itemize}

\section{Conexión con el resto del curso}
La clase 09 no se estudia aislada: se conecta con RPC, replicación, consistencia, observabilidad o resiliencia según corresponda. El objetivo pedagógico es que el estudiante pueda trasladar el concepto de una situación puntual a una familia completa de problemas distribuidos.

\section{Resultado de aprendizaje esperado}
Al finalizar la clase, el estudiante debería poder explicar el problema central, formular el modelo de fallas relevante, proponer una solución razonable y justificar el trade-off dominante.
