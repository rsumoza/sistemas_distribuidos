\begin{uaexercise}
Definir consenso.
\end{uaexercise}

\begin{uaanswer}
El problema de consenso consiste en que un conjunto de procesos distribuidos acuerden un único valor a partir de propuestas iniciales, bajo condiciones de asincronía y posibles fallas.

Formalmente, cada proceso $p_i$ propone un valor $v_i$ y eventualmente decide un valor $v$ tal que:
\begin{itemize}
\item Agreement: todos los procesos correctos deciden el mismo valor
\item Validity: $v$ fue propuesto por algún proceso
\item Termination: todo proceso correcto decide eventualmente
\end{itemize}
\end{uaanswer}

\begin{uaexercise}
Explicar FLP.
\end{uaexercise}

\begin{uaanswer}
El teorema FLP establece que en un sistema asíncrono con al menos una posible falla, no existe un algoritmo determinista que garantice la terminación del consenso.

La prueba se basa en:
\begin{itemize}
\item estados bivalentes (donde el resultado no está determinado)
\item un scheduler que retrasa mensajes críticos
\end{itemize}

Esto permite construir ejecuciones infinitas donde el sistema nunca decide.
\end{uaanswer}

\begin{uaexercise}
Probar intersección de mayorías.
\end{uaexercise}

\begin{uaanswer}
Sea $N$ el número total de nodos.

Una mayoría es cualquier conjunto de tamaño $> N/2$.

Sean $Q_1$ y $Q_2$ dos mayorías.

Supongamos que no intersectan:
\[
|Q_1| + |Q_2| \le N
\]

Pero:
\[
|Q_1| > N/2, \quad |Q_2| > N/2
\]

Entonces:
\[
|Q_1| + |Q_2| > N
\]

Contradicción.

Por lo tanto:
\[
Q_1 \cap Q_2 \neq \emptyset
\]
\end{uaanswer}

\begin{uaexercise}
Explicar Log Matching en Raft.
\end{uaexercise}

\begin{uaanswer}
La propiedad establece que si dos logs contienen una entrada con el mismo índice y término, entonces todos los elementos anteriores son idénticos.

Esto se garantiza porque:
\begin{itemize}
\item el líder solo añade entradas a partir de logs consistentes
\item los followers rechazan entradas inconsistentes
\end{itemize}
\end{uaanswer}
